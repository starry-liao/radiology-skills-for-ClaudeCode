# Grant architecture (NSFC / provincial / institutional)

Section-by-section: what each must accomplish. Order and names follow NSFC (国自然) conventions;
adapt for provincial/institutional. Confirm current required sections/limits against the official
申报指南.

| Section | Purpose | Reviewer asks |
|---|---|---|
| **题目 (Title)** | Question + approach + domain, concrete | Is it specific and scientific, not a slogan? |
| **摘要 (Abstract)** | The whole loop in one paragraph: need → question → aims → route → significance | Can I grasp the science and why it matters in 30 seconds? |
| **立项依据 (Background/rationale)** | Clinical need → state of the art → **gap** → this proposal | Is the gap real and current? Is the logic to the question tight? |
| **研究目标 (Objectives)** | What the project will establish | Are objectives concrete and bounded? |
| **研究内容 (Specific aims)** | The testable sub-questions and what each produces | Do all aims serve the key question? Independent yet coherent? |
| **关键科学问题 (Key scientific question)** | The single spine question | Is there exactly one, and is it answerable? |
| **技术路线 (Technical route)** | The closed loop from question to expected result | Is it complete, logical, with validation and contingencies? |
| **创新点 (Innovation)** | The specific, defensible increment | Is it concrete (not "novel"), and tied to the gap? |
| **可行性分析 (Feasibility)** | Why this team can do this now | Preliminary data? capability? data/ethics access? |
| **预期成果 (Expected outcomes)** | Outputs (papers, models, IP) realistically | Achievable in the period/budget? not over-promised? |
| **研究基础/工作条件** | Team track record, platform, prior work | Credible to deliver? |
| **年度计划 (Timeline)** | Aims across the funding period | Realistic sequencing? |

## Abstract template (loop in one paragraph)

```
针对[临床需求]，目前[现状与不足/gap]。本项目拟回答[关键科学问题]，假设[hypothesis]。围绕该问题，
开展[aim1]、[aim2]、[aim3]，采用[关键方法]，并通过[内部/外部/前瞻验证]验证。预期阐明[机制/规律]，
为[临床转化/应用]提供[依据]。本项目的创新在于[具体创新点]。
```

## 立项依据 logic chain (must be tight)

clinical need → what's known → **specific unanswered question (gap)** → why now (data/method
ready) → this proposal answers it. Each link cited and current (verify gaps live via
`radiology-frontier`/`radiology-search`; never fabricate citations).
